    7.Segment trees

    1. 线段树的性质

    一棵线段树存储一个区间范围，每个结点中存储附加属性。

    • 对于一个区间为 $[l,\ r)$ 的结点，其左右子树对应的区间分别为 $[l,\ mid)$ 和 $[mid+1,\ r)$;
    • 由于区间向下二分，兄弟结点间的区间大小的差值至多为 $1$，因此线段树或为满二叉树 (总元素数为 $2$ 的幂)，或为最后一层未满的满二叉树，树高为 $O(\lg n)$，通常根据根节点区间大小开 $4$ 倍空间；
    • 叶结点代表的区间是一个单点；
    • 对于线段树区间内的每个子区间 $i$，存在唯一确定的 $\log n$ 个结点，这些结点区间交集为空、并集为 $i$ 且 $n$ 值最小。

    2. 建树 🌳

     线段树和平衡树的一点不同是，其结构一旦建立便不可改变。
     当忽略最后一层的结点，线段树是一棵满二叉树，因此可用数组表示。每个结点的属性通常仅与左右子树有关，这里以区间和为例。

    nodes[idx] = nodes[idx * 2 + 1] + nodes[idx * 2 + 2];
}

    3. 查询与修改

    void modify(int idx, int l, int r, int point, int val) {
    if (l + 1 == r && l == point) {
        nodes[idx] = val; 
        return; 
    }
    int mid = l + (r - l) / 2;
    if (point < mid)
        modify(idx * 2 + 1, l, mid, point, val);
    else
        modify(idx * 2 + 2, mid, r, point, val);
    nodes[idx] = nodes[idx * 2 + 1] + nodes[idx * 2 + 2];
}

    给定线段树区间的一个子区间，查询该区间内所有结点的聚合属性信息。线段树的查询是一个遍历二叉树寻找子区间的过程，根据线段树的性质，可将查询分为三种情况：

    • 查询区间与当前区间无交集，则不再向下遍历并返回 0；
    • 查询区间包含了当前区间，则不再向下遍历并返回当前区间的属性；
    • 查询区间与当前区间有交集，则向左右依次递归继续查询，结束后返回二个查询结果的和。
    如果对向左右子树转移加以限制，则可以忽略不相交区间的情况。

    /*
 * [lc, rc): current interval; [ls, rs): interval to search
 */
int query(int idx, int lc, int rc, int ls, int rs) {
    if (lc >= ls && rc <= rs) {
        return nodes[idx];
    }
    int mid = lc + (rc - lc) / 2, res = 0;
    if (ls < mid) 
        res += query(idx * 2 + 1, lc, mid, ls, rs);
    if (rs > mid)
        res += query(idx * 2 + 2, mid, rc, ls, rs);
    return res;
}

    (2). 区间修改 + 区间查询

    如果需要修改某个区间内的值，即令 $[ls, rs)$ 的所有单点同时加上 $val$，那么这个区间对应的所有子区间结点的属性值都要更新，其复杂度无法接受。此问题可通过永久化标记解决，即为每个结点添加一个标记属性，区间修改时，只在对应根结点上做个标记。则：

    • 区间修改时，需要更新被标记结点的祖先结点的结点属性值；
    • 区间查询时，为使标记生效，查询结果 = 当前结点子区间标记计算 + 左右子树查询结果。

    void modify(int idx, int lc, int rc, int ls, int rs, int val) {
    if (lc >= ls && rc <= rs) {
        addsum[idx] += val;
        return;
    }
    nodes[idx] += (min(rs, rc) - max(ls, lc)) * val;
    int mid = lc + (rc - lc) / 2;
    if (ls < mid)
        modify(idx * 2 + 1, lc, mid, ls, rs, val);
    if (rs > mid)
        modify(idx * 2 + 2, mid, rc, ls, rs, val);
}

int query(int idx, int lc, int rc, int ls, int rs) {
    if (lc >= ls && rc <= rs) {
        return nodes[idx] + (rc - lc) * addsum[idx];
    }
    int mid = lc + (rc - lc) / 2;
    int res = (min(rs, rc) - max(ls, lc)) * addsum[idx];
    if (ls < mid)
        res += query(idx * 2 + 1, lc, mid, ls, rs);
    if (rs > mid)
        res += query(idx * 2 + 2, mid, rc, ls, rs);
    return res;
}

    4. 离散化

     当区间 $[l, r)$ 的范围很大，且其中的有效子区间很少，为节省空间并提升线段树的效率，需要对数据进行离散化。假设要存储的区间为 $[1, 100)$ 和 $[100, \mathrm{inf})$，则可以把这几个数分别映射到 $1、2、3$ 再通过线段树存储。
     需要注意的是，当线段树存储区间的形式为左闭右闭且需要离散化时，$(\mathrm{mid}, \mathrm{mid+1})$ 内的数据无法表示。
