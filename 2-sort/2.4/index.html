    4.Quicksort

    1. 快速排序的基本描述

    ① QUICKSORT(A, l, r):
     快速排序的基本思想是分治。

    void QuickSort(int A[], int left, int right) {
    if (left < right) {
        int i = partition(A, left, right);
        QuickSort(A, left, i - 1);
        QuickSort(A, i + 1, right); 
    }
}
    ② PARTITION(A, l, r):
    快速排序的核心是划分的过程，首先选取主元(pivot element)，通过主元再把排列划分为两部分。以下代码以 pivot=A[r] 为例。

    int partition(int A[], int left, int right) {
    int pivot = A[right];
    int i = left - 1;
    int j;
    for (j = left; j <= right - 1; j++) {
        if (A[j] <= pivot) {
            i++;
            swap(&A[i], &A[j]);
        }
    }
    swap(&A[i + 1], &A[right]);
    return i + 1;   
}

    在迭代过程中，数组被划分为四个区域，每一个区域都满足一定性质，我们称之为循环不变量。
    每一轮迭代开始时，对任意数组下标 k ，有：
     1. 若 l ≤ k ≤ i ，则 A[k] ≤ pivot。
     2. 若 i + 1 ≤ k ≤ j - 1 ，则A[k] > pivot。

    2. 快速排序的性能分析

    (1). 最坏情况

     当划分产生的两个子问题分别包含n-1个元素和0个元素时，$T(0)=\Theta(1)$，因此有： $T(n)=T(n-1)+\Theta(n)$，直观上来看，复杂度的结果为级数的和，即 $\Theta(n^2)$。

    (2). 平衡划分

     在可能最平衡的划分中，两个子问题的规模分别为 $\lfloor n/2 \rfloor$ 和 $\lceil n/2 \rceil-1$，于是有递归式：$T(n)=2T(n/2)+\Theta(n)$，解为 $T(n)=\Theta(nlgn)$。
     实际上，快排的平均运行时间更接近最好情况而非最坏情况。
     对于任何一种常数比例的划分，快排的时间复杂度都是 $O(nlgn)$。以 $T(n)=T(9n/10)+T(n/10)+cn$ 为例，画出递归树可得每层的代价都是 $cn$ ，其深度为 $log_{10/9}n$。

    (3). 随机化快速排序分析

    随机化快速排序，即主元随机选取 

    RANDOMIZED-PARTITION(A, left, right)    
    i = RANDOM(left, right)
    exchange A[right] with A[i]
    return PARTITION(A, left, right)
    推导前的准备：
    ① 当在元素个数为 n 的数组上运行 QUICKSORT 时，假设在 PARTITION 处的比较次数为 $X$，则 QUICKSORT 的运行时间为 $O(n+X)$。
     - 至多调用 PARTITION $n$ 次，调用时间为 $O(n)$ 加上迭代时间(比较次数)。
    ② 每个元素只可能与主元比较，因此对于任一对元素，它们至多比较一次
    ③ 一旦主元 $z_i<pivot<z_j$ 被选择，被划分的 $z_i$ 与 $z_j$ 不可能再比较(假设每个元素互异)

    推导：
     设 $X_{ij}= \begin{cases} 1& if\ z_i\ compares\ with\ z_j\\ 0& else \end{cases}$，$X= \sum_{i=1}^{n-1}\sum_{n}^{j=i+1} X_{ij}$
     则有 $$\begin{aligned} E(X)=E[\sum_{i=1}^{n-1}\sum_{n}^{j=i+1} X_{ij}]&=\sum_{i=1}^{n-1}\sum_{n}^{j=i+1}P(\lbrace z_i\text{与}z_j\text{比较}\rbrace) \\ &=\sum_{i=1}^{n-1}\sum_{n}^{j=i+1}P(\lbrace z_i\text{或}z_j\text{是集合}Z_{ij}中选出的第一个主元\rbrace) \\ &=\sum\sum(\frac{1}{j+i-1}+\frac{1}{j+i-1})=\sum\frac{2}{j+i-1} \\ &< \sum_{i=1}^{n-1}\sum_{n}^{k=1} \frac{2}{k}=O(nlgn) \end{aligned}$$
     故得出结论：使用随机化快速排序，在输入元素互异的情况下，快排的期望运行时间为 $O(nlgn)$。

    (4). 快排特点

    ① 期望复杂度为 $\Theta(nlgn)$，与堆排序、归并排序相同，但其常数更小，速度更快
    ② 快排是不稳定的

    3. 三数取中划分

    从子数组中随机选取三个元素，取其中位数作为主元。 

    int median3(int A[], int left, int right) {
    int center = (left + right)/2;
    if (A[left] > A[right]) {
        swap(&A[left], &A[right]);
    }
    if (A[left] > A[center]) {
        swap(&A[left], &A[center]);
    }
    if (A[center] > A[right]) {
        swap(&A[center], &A[right]);
    }   
    swap(&A[center], &A[right - 1]);
    return A[right - 1];        
}
    在代码上，要注意当问题规模为2时要单独考虑。 
    void QuickSort(int A[], int left, int right) {
    if (right > left + 1) {
        int pivot = median3(A, left, right);
        int i = left;
        int j = right - 1;
        while (1) {
            while(A[++i] < pivot){ } // less or equal cannot fit that the pivot needn`t move
            while(A[--j] > pivot){ } //   for i should stop at the pivot
            if(j > i)
                swap(&A[i], &A[j]);
            else
                break;
        }
        swap(&A[i], &A[right - 1]);
        QuickSort(A, left, i-1);
        QuickSort(A, i+1, right);   
    } else {
        if (A[right] < A[left]) {
            swap(&A[right], &A[left]);
        }
    }
}
