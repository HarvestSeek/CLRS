    6.Median and order statictics

    1. 最大值最小值

     对一个给定的含有 $n$ 个元素的集合，找出其中的最大(小)值需要 $n-1$ 次比较，且这个值也是比较次数的下界。
     某些场景需要同时找出最大值和最小值，按照一般的想法则需要 $2n-2$ 次比较。事实上，至多通过 $3\lfloor n/2 \rfloor$ 次比较就可以实现。具体的做法是成对地处理元素，先比较这两个元素，然后其中较小的与已知的最小值比较，较大的与已知的最大值比较，这样每两个元素就只需要三次比较。
     对于奇数个元素，需要 $3\lfloor n/2 \rfloor$ 次比较；对于偶数个，需要 $3n/2-2$ 次比较，所以至多需要 $3\lfloor n/2 \rfloor$ 次比较。

    2. 顺序统计量选择

    (1). 期望时间为线性的算法

     类似于快速排序的思想，使用 RANDOMIZED-PARTITION(A, left, right) 对数组进行划分，和快排不同的是只向一边递归。
     RANDOMIZED-SELECT(A, i, left, right): 在数组 A[left..right] 中找出第 i 小的元素。

    int RandomizedSelect(int A[], int index, int left, int right) {
    if (left == right) {
        return A[left];
    }
    int i = RandomizedPartition(A, left, right);
    if (i + 1 == index) {
        return A[i];
    } else if (i + 1 < index) {
        return RandomizedSelect(A, index, i + 1, right);
    } else {
        return RandomizedSelect(A, index, left, i - 1);
    }
}

int RandomizedPartition(int A[], int left, int right) {
    int pivotI = rand() % (right - left + 1) + left;
    swap(&A[pivotI], &A[right]);
    int i, j;
    for (i = left - 1, j = left; j < right; j++) {
        if (A[j] <= A[right]) {
            swap(&A[++i], &A[j]);
        }
    }
    swap(&A[++i], &A[right]);
    return i;
}

    (2). 最坏时间为线性的算法

     此算法仍然通过划分的思想实现，但与上面的思路上不同的是，每次都保证实现一个较好的划分，把主元作为参数传递给划分函数。
     SELECT(A, i, left, right): 通过以下步骤，在数组 A[left..right] 中找出第 i 小的元素。
     ① 将数组划分为 $\lfloor n/5 \rfloor$ 组，每组 $5$ 个元素，至多还有一组有 $n\ mod\ 5$ 个元素。
     ② 寻找 $\lceil n/5 \rceil$ 组中每组的中位数：插入排序
     ③ 递归调用 SELECT，找到 ② 中确定的 $\lceil n/5 \rceil$ 个中位数的中位数 $x$。(如果元素个数为偶数，约定取下中位数)
     ④ 以 $x$ 为主元，对数组进行划分。令 $x$ 是第 $k$ 小的元素，则低区元素个数为 $k$，高区元素个数为 $n-k$。
     ⑤ 如果 $i==k$，则返回 $x$；如果 $i < k$，则在低区递归寻找第 $i$ 小的元素；如果 $i>k$，则在高区寻找第 $i-k$ 小的元素。

    int Select(int A[], int index, int left, int right) {   
    if (right == left) {
        return A[left];
    }
    int len = right - left + 1;
    int j;
    for (j = 0; j < len / 5; j++) {
        InsertionSort(A, left + j * 5, left + j * 5 + 4 );
        swap(&A[left + j], &A[left + j * 5 + 2]);
    }
    if (j * 5 < len) {
        InsertionSort(A, left + j * 5, left + len - 1);
        swap(&A[left + j], &A[left + (len - 1 - j * 5) / 2]);
        j++;
    }
    int x = Select(A, j / 2, left, left + j - 1);
    int k = Partition(A, left, right, x);
    if (k - left + 1 == index) {
        return x;
    } else if (k - left + 1 > index) {
        return Select(A, index, left, k);
    } else {
        return Select(A, index - (k - left + 1), k + 1, right);
    }
}

void InsertionSort(int A[], int left, int right) {
    int i, j;
    for (i = left + 1; i <= right; i++) {
        int t = A[i];
        for (j = i - 1; A[j] > t && j >= left; j--) {
            A[j + 1] = A[j];
        }
        A[j + 1] = t;
    }
}

int Partition(int A[], int left, int right, int pivot) {
    int i, j;
    for (i = left - 1, j = left; j <= right; j++) {
        if (A[j] <= pivot) {
            swap(&A[++i], &A[j]);
        }
    }
    return i;
}
     下面证明它的时间复杂性，假设元素互异。首先，要得到递归方程，就要知道第 ⑤ 步递归最多处理多少个元素。在 ② 中划分的 $\lceil n/5 \rceil$ 个组中，至少有 $\lceil \frac{1}{2} \lceil n/5 \rceil \rceil$ 个组包含大于 $x$ 的元素，其中除了 $x$ 本身的组和未被整除的组，剩下的组均贡献了三个。所以大于 $x$ 的元素至少有

    $$ 3(\lceil \frac{1}{2} \lceil \frac{n}{5} \rceil \rceil - 2) \geq \frac{3n}{10}-6$$

    个。由于 $x$ 是下中位数，小于 $x$ 的分析也同理。则第 ⑤ 步中处理元素的个数至多为 $7n/10+6$ 个。步骤 ①②④ 需要 $O(n)$ 时间，其中 ② 是对 $O(1)$ 空间使用 $O(n)$ 次插入排序。步骤 ⑤ 需要 $T(7n/10+6)$ 时间。于是可得递归式，并使用代入法假设存在相应的常数 $c$ 和 $n_0$：

    $$ \begin{aligned} T(n) & \leq T(\lceil n/5 \rceil)+T(\frac{7n}{10}+6)+O(n) \\ & \leq \frac{9cn}{10}+7c+an \\ & = cn+(-\frac{cn}{10}+7c+an) \end{aligned} $$

    因此，只需要 $-cn/10+7c+an \leq 0$，该算法最坏时间复杂度为线性。

    3. 两个有序数组的中位数

     给定两个有序数组 $X[0..m-1]$ 和 $Y[0..n-1]$，要求找出两个数组的 $m+n$ 个元素的中位数，且复杂度为 $O(log(m+n))$。
     如果 $m==n$，设 $X$ 和 $Y$ 的中位数分别为 $x$ 和 $y$，可以证明两个数组的中位数一定位于区间 $[x,\ y]$ 内，这样就很容易根据问题定义的状态找到子问题向下递归。
     但 $m$ 与 $n$ 的关系不确定，无法简单的向下划分，这时需要转换思路。这个问题等价于找到对数组 $X$ 和 $Y$ 的一种划分：

    $$ \begin{aligned} \lbrace X[0],\ ...,X[i-1] \ &|\ X[i],\ ...,X[m-1] \rbrace \\ \lbrace Y[0],\ ...,Y[j-1] \ &|\ Y[j],\ ...,Y[n-1] \rbrace \\ i \in [0,\ m],\ & j \in [0,\ n] \end{aligned} $$ 这种划分要满足： $$ \begin{cases} i+j=m-i+n-j\ (or\ i+j=m-i+n-j+1)& \\ X[i-1]\leq Y[j] \ \ and \ \ Y[j-1] \leq X[i]& \end{cases} $$  这里当 $m+n$ 为奇数，令左边的部分多一个元素，方便统一公式。且 $m$ 必须小于等于 $n$，否则 $j$ 可以为负。
     经过这些步骤后，原问题可以转化为以下的搜索问题：在区间 $[0,\ m]$ 中找到 $i$，使得 $j=(m+n+1)/2-i$ 且 $X[i-1]\leq Y[j]$ 且 $Y[j-1] \leq X[i]$。根据这些条件，在 $[0,\ m]$ 内使用二分搜索。最后在代码中要注意取到临界值条件的判断：
     ① 当 $i==0$ 且 $Y[j-1]\leq X[i]$ 时，$i$ 的位置正确，在另一个条件中注意判断 $i!=0$。
     ② 当 $i==m$ 且 $X[i-1]\leq Y[j]$ 时，$i$ 的位置正确，在另一个条件中注意判断 $i!=m$。
     ③ $j$ 可以取到临界值，当且仅当两数组长度相等且 $i$ 取到了临界值。

    double FindMedianInSortedArrays(int X[], int Y[], int xLen, int yLen) {
    if (xLen > yLen) {
        int *tA = X;
        int tN = xLen;
        X = Y;
        xLen = yLen;
        Y = tA;
        yLen = tN;
    }
    int iLeft = 0, iRight = xLen;   
    while (iLeft <= iRight) {
        int i = (iLeft+ iRight) / 2;    
        int j = (xLen + yLen + 1) / 2 - i;
        if (i != xLen && Y[j - 1] > X[i]) {
            iLeft = i + 1;
        } else if (i != 0 && X[i - 1] > Y[j]) {
            iRight= i - 1;
        } else {
            int maxLeft;
            if (i == 0) {
                maxLeft = Y[j - 1];
            } else if (j == 0) {
                maxLeft = X[i - 1]; 
            } else {
                maxLeft = max(X[i - 1], Y[j - 1]);
            }
            if ((xLen + yLen) & 1 == 1) {
                return maxLeft;
            }
            int minRight;
            if (i == xLen) {
                minRight = Y[j];
            } else if (j == yLen) {
                minRight = X[i];
            } else {
                minRight = min(X[i], Y[j]);
            }
            return (maxLeft + minRight) / 2.0;
        }
    }
    return -INF;
}
